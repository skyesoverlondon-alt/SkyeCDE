SOC2-ish Controls Summary

Customer-facing controls summary aligned to SOC 2 themes (Security, Availability, Confidentiality, Privacy).

Security

Authentication
HttpOnly secure cookies + CSRF; verified email; optional SSO (OIDC/SAML) and SCIM provisioning.
Encryption
Messages/attachments are stored ciphertext-only. Web submit encrypts client-side; inbound import encrypts immediately for storage. Optional KMS mode supports managed key escrow.
Access Control
Organizations + RBAC; audit trail is immutable during retention.
Bot Defense
IP-hashed rate limiting + optional Cloudflare Turnstile on public entry points.
Security Headers
Strict CSP with no unsafe-inline scripts/styles; clickjacking protection; HSTS; no-sniff; permissions policy.

Availability

Hosting
Netlify Functions + CDN with Postgres (Neon). Health endpoint available for monitoring.
Backups
Backups are provided by the database provider; recommended daily backups + PITR if available.
SIEM Outbox
Audit events enqueue to an outbox and drain on schedule with retry/backoff.

Confidentiality & Privacy

Content
Notifications never include plaintext message content; subjects/bodies are encrypted together at rest.
Metadata
Sender identifiers and timestamps are stored plaintext for operational use.
IP Handling
Rate limiting stores only salted IP hashes (no raw IP persistence).
