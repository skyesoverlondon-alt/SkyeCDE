SOL Mobile Detailing Network Setup

This hub now runs on Neon/Postgres for operational data, Stripe for checkout, and Netlify Functions for session-backed APIs.

Required runtime pieces: DATABASE_URL, STRIPE_SECRET_KEY, and SITE_URL or URL.

Environment Variables

  • DATABASE_URL: Neon/Postgres connection string.
  • STRIPE_SECRET_KEY: Stripe secret key for checkout session creation and payment verification.
  • SITE_URL or URL: Public base URL used for Stripe success/cancel redirects.
  • DEFAULT_DEPOSIT_CENTS: Optional override for the initial deposit amount.
  • SESSION_COOKIE_SECURE: Optional. Set to false for local HTTP testing only.
  • ADMIN_EMAILS: Optional comma-separated list of emails that should automatically receive the admin role.

Database Bootstrapping

No manual table creation is required. The first function call will create the schema automatically, including:

  • hub_users and hub_sessions
  • detailer_intake and detailers
  • service_requests, jobs, and subscriptions
  • job_updates and job_messages

Operational Flow

  • /index.html: detailer intake form writes to detailer_intake.
  • /admin.html: admin reviews submissions, approves detailers, sets roles, and creates jobs.
  • /request.html: client creates a service_requests row and is redirected to Stripe Checkout.
  • /success.html: checkout success finalizes payment, creates the job, and optionally starts a subscription record.
  • /client.html and /caregiver.html: session-backed job updates and messaging.

Local Development

npm install
netlify dev

Use a local Postgres-compatible DATABASE_URL or your Neon development branch. For local non-HTTPS sessions, set SESSION_COOKIE_SECURE=false.

Verification

  • Submit a detailer intake and verify it appears in admin.
  • Approve the intake and verify a hub_users record plus a detailers record are created.
  • Create a client account from /request.html, submit a request, and complete a Stripe test checkout.
  • Confirm the success flow creates a jobs record and that both portals can see messages and updates.
