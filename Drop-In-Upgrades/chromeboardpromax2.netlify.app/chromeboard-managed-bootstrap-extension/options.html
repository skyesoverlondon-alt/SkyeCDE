ChromeBoard Managed Bootstrap Setup

1) Set the ChromeBoard URL

For most users, put your live ChromeBoard site URL here — usually your Netlify URL or custom domain. Only use a local URL during development. Enterprise policy exists only if an admin wants to prefill the same destination for many managed users.

1b) Which URL should I actually use?

Usually: yes, use your Netlify URL. Example: https://your-board.netlify.app/.

Use a localhost URL only while developing on your own machine. If you are not doing managed enterprise rollout, you can ignore the policy wording and just save the live board URL here.

3) Reality check

This extension can read enterprise-managed values from chrome.storage.managed, import installed apps that Chrome exposes through chrome.management.getAll(), and import bookmarks/tabs. It still cannot magically read the private native launcher pin list on an unmanaged Chromebook because Chrome does not publish a normal read API for that OS surface.

That is why this build uses the same enterprise lane that admins already use to force-install or pin apps, then mirrors that seed into ChromeBoard.
